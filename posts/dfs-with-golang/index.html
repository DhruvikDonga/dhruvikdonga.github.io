So we will be solving a hackerrank problem on Graph Data structures to understand how to implement this kind of problems in golang by this blog we will understand maps, slices, recursion, global variables, for loops in golang. Also we will understand what is connected component in graph and how to use the logic to ace in these type of problems .

Let’s begin ✏

Hope you have read the question so breaking it down we will build a logic that we will implement in golang . So the question basically states that they have astronauts from different countries and now you have to return total no of pairs of astronauts are possible which have to belong to different countries so this is a condition.

Analyzing the inputs we understand it is a graph question as the edges are given in it . Now with a example we will visualize what exactly we need to do:-

Input:-
10 7
0 2
1 8
1 4
2 8
2 6
3 5
6 9

So the astronauts are 10 (0 to 9) and 7 edges are given let’s build a graph out of it 🖊

This graphs contains components which are not connected there are 7 nodes connected one another , 2 nodes to 1 component and 1 node to no one . So there are 3 countries one with 7 astronauts others with 2 and 1 . Forming the pairs we get 7X2 + 7X1 + 2X1 :- 23 pairs

We will be using the depth first search to travel through this graph and calculating the connected components slice array which are countries and no of astronauts in it.

Golang solution 👨🏻‍💻 

package main

import(
    "fmt"
)
var visited [100001]int //to keep record of the astronauts who are alreay counted
var astrocnt int //to count the no of astronauts of a country
var graph = make(map[int][]int) //to construct a graph of astronauts pairing with eachother
var country []int //this slice will store the astronauts present in a country


//depth first search recursion approach to get all the astronauts connected to a component


func dfs(astr int){
    visited[astr]=1
    
    //if astronaut is connected then count will add that number and dfs will continue
    for _,v := range graph[astr] {
        if visited[v]==0{
            astrocnt +=1
            dfs(v)
        }
    }
    
}

func main(){
    var n,p int //no of astronauts and pairs
    
    fmt.Scan(&n)
    fmt.Scan(&p)
    
    for i :=0 ; i<p ;i++ {
        var a,b int
        fmt.Scan(&a)
        fmt.Scan(&b)
        
        //graph construction
        graph[a] = append(graph[a],b)
        graph[b] = append(graph[b],a)
    }
    //fmt.Print(graph)
    
    //for loop will check for astronauts visited or not if not then a connected component will start
    for i:=0;i<n;i++ {
        if visited[i]==0{
            astrocnt=1
            dfs(i)
            country = append(country,astrocnt) //slice country storing no of astronauts you got in dfs recursion
        }
    }
    rem,pairs:= 0,0
    //fmt.Println(country)
    /**
    
    logic of for loop with example:-
    country = [7 2 1]
    so for now pairs we get is 7*2 + 7*1 + 2*1 which is 23
    but the time complexity will increase in doing this
    Cool Logic:-
    we know that total no of astronauts are given and this array is country wise distribution of it and it matters only what the product is of the astronaut of that country and remaing ones :- 
    1st country
    now :- 7*2 + 7*1 = 14+7 = 21
    by the logic above :- 7 * (10-7) :- 21
    
    2nd country:-
    7 astros are taken in rem new 2 are left so
    2*1 = 2 and 2+21 = 23
    logic in for loop :- 2 * (10-(7+2)) = 2
    **/
    
    for _,v := range country {
        rem = rem+v
        pairs = pairs+ v*(n-rem)
    }
    fmt.Println(pairs)
    
}

Conclusion

So in this code we have declared a graph which is map of slices and the slices stores the nodes which are connected to it . DFS function will traverse through the input node and to it’s connected ones . So Depth first search goes to depth of the node to it’s end and it uses stack datastructure but since Recursion process in functions is also stack so we are good with this

The country slice is dynammic array which holds no of astronauts count in each connected component.
